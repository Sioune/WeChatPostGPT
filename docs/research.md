# 微信公众号文章模板与格式调研

调研日期：2026-05-06

## 关键约束

公众号图文的可复制排版，本质上是把内容降级成微信编辑器可保留的静态 HTML。

- 草稿/图文内容字段支持 HTML，但要求内容长度受限，并会去除 JS；图片 URL 需要来自微信图文图片上传接口，外部图片可能被过滤。参考 WxJava 对微信草稿接口文档的摘录说明：https://javadoc.io/static/com.github.binarywang/weixin-java-mp/4.2.6.B/me/chanjar/weixin/mp/api/impl/WxMpDraftServiceImpl.html
- 实操上应避免 `<script>`、`<style>`、`iframe`、事件属性、复杂定位、媒体查询和动画；应把样式写到元素的 `style` 属性里。参考 HTML/CSS 支持情况整理：https://www.axtonliu.ai/newsletters/ai-2/posts/wechat-article-html-css-support
- 输出结构应优先使用 `section`、`p`、`blockquote`、`ul/ol/li`、`table`、`img`、`strong`、`em`、`code` 等静态标签。

## 主流工具能力

### WeMD

WeMD 强调微信公众号排版、本地优先、精准预览、主题定制和 CSS 双模式。它的启发是：排版智能体应保护创作者文本隐私，并把模板作为可调整的系统，而不是一次性样式。  
来源：https://wemd.app/

### WeDown

WeDown 的关键点是 Markdown 写作、实时预览、CSS 内联算法、多主题和自定义主题系统。它说明“复制到公众号后稳定”的核心是内联样式，而不是依赖外部 CSS。  
来源：https://wedown.app/

### mdnice

mdnice 以 Markdown 结构化写作为基础，再通过主题渲染生成兼容公众号的文章。它的启发是：内容结构越清晰，模板套用越自然，主题也更容易形成完整风格。  
来源：https://product.mdnice.com/

### MD2WE

MD2WE 展示了更完整的创作流：主题风格、代码主题、字号、复制 HTML、下载 HTML、AI 创作助手、模板片段、图片上传和推送草稿。它的启发是：排版工具应同时覆盖生成、复制、素材与发布前检查。  
来源：https://md2we.com/

### MPMD

MPMD 强调丰富排版样式、实时预览、一键复制到公众号、浏览器插件和图片托管。它的启发是：复制链路和图片链路是公众号排版体验的关键闭环。  
来源：https://mpmd.pages.dev/

## 模板设计结论

一套合格的公众号模板至少要包含这些稳定部件：

- 文头系统：类型标签、主标题、导语、分隔线或色块。
- 正文章节：编号、标题容器、正文行高、段间距。
- 强调系统：引用、关键句、加粗、行内代码、链接。
- 信息块：列表、表格、步骤卡、数据块。
- 媒体处理：图片宽度、圆角、说明和素材库提醒。
- 收尾系统：轻量元信息或结语。

因此，本项目没有把模板做成“标题颜色 + 正文字号”的组合，而是把模板定义成完整的 v2 版式系统：`palette + rhythm + design tokens + renderer`。其中 `design` 明确描述 hero、heading、quote、list、image、divider 六组结构令牌，让 AI 学到的模板能真正改变整篇文章的视觉骨架。

## 智能体策略

本项目把智能体拆成两条能力分支：

1. 模板生成/学习分支
   - 从参考 HTML 中抽取颜色、字号、行高、圆角、边框与关键词。
   - 推断模板 family：科技、数据、活动、人文、品牌、教育、商业、洞察。
- 生成可复用 v2 模板并保存到浏览器本地。

2. 内容套用分支
   - 分析标题和正文关键词、段落数、标题数、数字密度、代码信号。
   - 自动选择匹配模板。
   - 若匹配分不足，基于内容类型生成新模板。
   - 当用户明确选择“AI 决策 + 内容驱动生成”时，不再沿用已有模板，而是要求 AI 原创一套 v2 版式令牌；本地渲染完整效果后立即沉淀为新模板。
   - 输出静态、内联样式 HTML，并提供微信兼容检查。

## AI 接入策略

实施时采用“AI 决策，本地渲染”的架构：

- AI 不直接生成公众号 HTML，而是通过 OpenAI Responses API 返回结构化 JSON 排版计划。
- JSON Schema 约束字段包括：内容分析、模板选择原因、密度、模板 family、颜色令牌、节奏令牌、design 版式令牌和建议。
- 本地渲染器继续负责生成 inline style HTML，避免模型输出不稳定标签或微信不兼容样式。
- 另增“AI 原生创意排版”作为高自由度模式：AI 直接返回完整 inline HTML，本地只做危险标签/事件属性清洗、兼容检查和复制交付。它牺牲一部分模板可复用性，换取更高的内容贴合度和视觉发挥。
- API 不可用、未配置 Key 或请求失败时，自动退回本地规则引擎。
- API Key 只放在本地 Node 代理服务中，不暴露给浏览器。
- 新版也支持在页面右上角配置 OpenAI 兼容 BaseUrl、API Key 和模型列表。页面配置保存在本机浏览器，并在请求时交给本地代理服务调用模型；`.env` 仍作为可选后备配置。

OpenAI 官方文档说明，Responses API 可生成文本/JSON 输出，Structured Outputs 可以让模型响应匹配 JSON Schema。参考：https://developers.openai.com/api/docs/guides/structured-outputs 和 https://platform.openai.com/docs/api-reference/responses/object

## 当前实现边界

- 这是本地原型，不会直接登录或操作微信公众号后台。
- “一键复制”指复制富文本，然后到公众号后台正文区粘贴。
- 公众号正式发布前，外链图片仍建议上传到微信素材库。
- 浏览器在 `file://` 页面可能限制高级剪贴板 API，应用已提供选中预览区复制的 fallback。
- URL 模板学习依赖本地后端抓取网页源码；微信公众号链接会优先验证并提取 `#js_content` 正文区域，再返回提取模式、正文长度和样式数量。若只拿到验证页、分享壳或无法定位正文区域，系统会停止学习并提示人工导入，避免用错误页面生成模板。
- 模板学习发给 AI 前会压缩为版式样本，保留结构、inline style、图片位置和少量文字片段，降低大体积富文本触发网关超时的概率。
- 模板学习不再使用本地规则兜底；AI 未启用或请求失败时会明确提示配置/网络问题，避免生成与参考文章弱相关的伪模板。
- 上传图片会保存到本地 IndexedDB 图片素材库，正文只保留短引用并参与排版，AI 请求会省略 base64 图片数据和本地图片 ID；正式发布时仍建议替换成微信素材库 URL。
