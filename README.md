# WeChatPostGPT

一个 AI 辅助的微信公众号智能排版体。输入标题和正文后，它会让 AI 生成结构化排版决策，再由本地渲染器输出可复制到微信公众号后台的内联 HTML 富文本。

## 使用方式

### AI 增强模式

1. 运行：

```bash
npm start
```

如果 Windows PowerShell 提示 `npm.ps1 cannot be loaded`，使用：

```bash
npm.cmd start
```

2. 打开 `http://localhost:5173`。
3. 在右上角「AI 配置」中填写 BaseUrl 和 API Key，点击「测试连接」。
4. 看到「测试链接成功！」后，在模型类型下拉框选择模型并保存。
5. 输入文章后点击「智能排版」。

如果 5173 被其他项目占用，可以在 `.env` 中改成例如 `PORT=5888`，然后重新运行 `npm start`。

### 本地规则模式

也可以直接用浏览器打开 `index.html`。这种方式不会调用 AI，只使用本地规则和模板渲染。

1. 输入文章标题和正文。
2. 点击「智能排版」，或等待自动刷新预览。
3. 点击「复制富文本」。
4. 打开 `https://mp.weixin.qq.com/` 的图文编辑器，在正文区域粘贴。

如果浏览器阻止了富文本剪贴板 API，工具会自动退回到选中预览区复制。仍失败时，可以手动选中右侧预览区后复制。

## 已实现能力

- AI 决策：后端通过 OpenAI Responses API 请求 v2 结构化 JSON，而不是让模型直接生成 HTML。
- 8 套整体式公众号模板：深度研究、商业简报、科技极简、人文随笔、活动官宣、知识卡片、品牌故事、数据报告。
- 内容分析：识别文章类型、语气、章节密度、代码/数字信号和阅读时长。
- 自动选模：AI 优先从模板库选择合适模板；API 不可用时退回本地打分。
- 内容驱动生成：AI 可生成新的配色、节奏和 `design` 版式令牌，覆盖文头、标题、引用、列表、图片和分隔系统；API 不可用时退回本地生成。
- 创意排版沉淀：当排版引擎为“AI 决策 + 本地渲染”、模板模式为“内容驱动生成”时，AI 必须生成新模板；右侧立即展示完整渲染效果，并自动把本次版式学习为可复用模板。
- 模板学习：支持直接粘贴富文本、输入 URL、上传 HTML 文件或手动粘贴 HTML；AI 总结视觉 DNA 并生成 v2 模板 schema。
- URL 学习：输入参考文章 URL 后，本地后端会抓取页面源码并自动进入模板学习流程。
- 模板管理：模板库支持按名称/描述/标签搜索，按风格和来源筛选，并可删除非内置模板。本地最多保存 200 套用户模板。
- 图片排版：正文侧支持上传图片并插入短引用图片块，可控制宽度、边框/卡片/海报样式和图片说明。
- 微信兼容输出：生成内容只依赖静态 HTML 与 inline style，并提供兼容检查。
- 一键复制：支持 `text/html` 富文本复制，并提供 HTML 源码复制。

## 调研结论摘要

调研见 [docs/research.md](./docs/research.md)。核心判断是：公众号排版工具的稳定路线不是复杂网页能力，而是把结构化内容渲染成静态、内联样式的富文本。成熟工具通常围绕 Markdown 写作、主题模板、实时预览、CSS 内联、一键复制/分发、图片托管和本地保存展开。

## 文件说明

- `index.html`：应用界面。
- `styles.css`：工作台样式，不会复制到公众号正文。
- `app.js`：前端智能体逻辑、模板库、模板学习、渲染和复制。
- `server.js`：本地静态服务器和 OpenAI API 代理，保护 API Key 不暴露到浏览器。
- `docs/research.md`：调研记录和设计依据。

## AI 配置

推荐在页面右上角「AI 配置」中设置：

- BaseUrl：例如 `https://api.openai.com/v1`，或你的 OpenAI 兼容服务地址。
- API Key：只保存在本机浏览器 `localStorage`，不会写入代码或日志。
- 模型类型：点击「测试连接」后会通过 `/models` 拉取并下拉展示。

仍保留 `.env` 作为可选后备配置：`OPENAI_BASE_URL`、`OPENAI_API_KEY`、`OPENAI_MODEL`。页面配置优先级高于 `.env`。

服务端会优先调用 Responses API；如果自定义 BaseUrl 不支持 `/responses` 且返回 404/405，会自动退到 Chat Completions API。Structured Outputs 用于约束 AI 只返回排版计划，模板 schemaVersion 当前为 `2`。

## 图片与 URL 注意事项

- URL 抓取需要通过 `npm start` 启动本地后端；直接打开 `index.html` 时浏览器无法跨站抓取网页源码。
- 后端允许抓取较大的参考页面，并会自动移除脚本、埋点、视频、SVG 等非排版内容，只保留适合模板学习的 HTML/CSS 片段。
- 微信公众号 URL 会优先提取真实正文区域 `#js_content`，并返回提取模式、正文长度和样式数量；如果只抓到验证页、分享壳或无法定位正文区域，会停止学习并提示改用富文本粘贴或上传 HTML。
- 有些公众号文章或网站会限制非浏览器抓取，失败时可以在浏览器里选中文章正文复制为富文本，或把网页另存为 HTML 后上传/粘贴再学习。
- 模板学习必须启用 AI。如果 `OPENAI_API_KEY` 未配置、网络不可用或模型请求失败，工具会提示“访问AI大模型失败，请检查配置或网络”，不会再退回本地生成误导性模板。
- 模板学习发给 AI 前会把参考富文本压缩为版式样本，保留结构、inline style、图片位置和少量文字片段，避免大段正文或 base64 图片导致模型网关超时。
- 上传图片会先压缩并保存到浏览器本地 IndexedDB 图片素材库，正文里只插入类似 `![图注](wechatpostgpt:image:img_xxx){width=90;style=framed;caption=图注}` 的短引用。
- 如果正文里已经有很长的 base64 图片，可以点击「收纳正文 base64」，工具会把它们转存进本地图片素材库，并把正文替换成短引用。
- 公众号正式发布前，建议把图片上传到公众号素材库，并用素材库图片地址替换本地短引用或 data URL。
- AI 排版请求会自动省略正文里的 base64 图片数据和本地图片素材 ID，只保留“这里有图片”的结构信号，避免把大图发送给模型。
